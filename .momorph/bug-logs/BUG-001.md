# BUG-001: All Kudos feed does not refresh after creating a new kudo

**Date:** 2026-03-19
**Status:** Fixed

---

## Summary

After submitting a new kudo via the Viáº¿t Kudo form, the All Kudos feed does not display the newly created kudo. `router.refresh()` is called correctly in `WriteKudoProvider.handleSuccess`, which causes the Next.js Server Component to re-run and pass fresh `initialData` to `AllKudosFeed`. However, `AllKudosFeed` is a Client Component that initializes its local state from `initialData` only once on mount â€” subsequent prop changes are ignored by React's `useState`.

---

## Root Cause

`AllKudosFeed.tsx` lines 14â€“16 use `useState(initialData.*)` to initialize `kudos`, `page`, and `hasMore`. React's `useState` only reads the argument on the **first render**; it does not sync with prop changes on subsequent renders. When `router.refresh()` triggers a Server Component re-render with new data, the Client Component receives the updated `initialData` prop but its local state remains unchanged.

---

## Fix Applied

**Option chosen:** `useEffect` to sync state when `initialData` changes

Added a `useEffect` with `[initialData]` dependency in `AllKudosFeed.tsx` that resets `kudos`, `page`, and `hasMore` whenever the server sends new data. This fires after `router.refresh()` delivers fresh `initialData` from the Server Component.
