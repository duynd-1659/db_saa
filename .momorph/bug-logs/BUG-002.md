# BUG-002: FAB "Thể lệ" button does not open rules panel

**Date:** 2026-03-19
**Status:** Fixed

---

## Summary

The "Thể lệ" button in `FabWidget` was rendered as a `<Link href="/awards-information">` using `next/link` (without locale prefix) while its `onClick` handler correctly calls `openPanel()` to open the `RulesPanelDrawer`. The conflicting element type caused unreliable behavior — the panel opening could be overridden by spurious navigation to a non-existent route. Fixed by replacing the `<Link>` with a `<button>` since the sole intent is to open the rules panel drawer.

---

## Root Cause

Two issues combined:
1. Element was `<Link>` but should be `<button>` — `handleRules` calls `openPanel()` (RulesPanelDrawer), not navigation. Using `<Link>` caused both the panel to open AND a navigation event to fire.
2. The `href="/awards-information"` used `import Link from 'next/link'` instead of `import { Link } from '@/i18n/navigation'`, meaning the bare path `/awards-information` has no locale prefix and would 404 (actual route is `/[locale]/awards-information`).

---

## Fix Applied

**Option chosen:** Replace `<Link>` with `<button type="button">`

Changed the "Thể lệ" element from `<Link href="/awards-information" onClick={handleRules}>` to `<button type="button" onClick={handleRules}>`. Removed the now-unused `import Link from 'next/link'`.

---

| `src/components/ui/FabWidget.tsx` | Replaced `<Link>` with `<button type="button">` for "Thể lệ"; removed unused `next/link` import |

## Notes

- The `RulesPanelDrawer` + `RulesPanelProvider` system is the correct handler for "Thể lệ" — no navigation needed.
- If navigation to `/awards-information` is ever needed elsewhere, use `import { Link } from '@/i18n/navigation'` (as done in `CTAButtons.tsx`).
